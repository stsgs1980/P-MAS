# Анализ UI графов/нод для редизайна страницы Hierarchy

## 1. Что не так с текущей реализацией

Текущий `agent-hierarchy.tsx` (3511 строк) реализует:

- **Radial layout** — агенты расставлены по концентрическим кольцам (8 колец = 8 групп: Стратегия→Тактика→Контроль→Исполнение→Память→Мониторинг→Коммуникация→Обучение)
- **Grid layout** — простая сетка 4×N
- **SVG-рендеринг** с canvas-анимациями (частицы, контурные линии фона)
- **6 типов связей**: command, sync, twin, delegate, supervise, broadcast
- **Побочные эффекты**: частицы вдоль связей, пульсации, glow-эффекты, контекстное меню

### Проблемы:

1. **Хаос при 26+ нодах** — радиальная схема без force-directed укладки даёт наложение
2. **Нет иерархии** — кольца по roleGroup не отражают реальную подчинённость (parentId)
3. **Перегруз эффектами** — анимированные частицы, glow, пульсации = визуальный шум
4. **Слабая интерактивность** — нельзя перетаскивать ноды, нет real-time layout
5. **Два режима (radial/grid)** не дают полноценной навигации по иерархии

## 2. Паттерны из Pinterest и индустрии

### 2.1 Node-based UI (Blender, Unreal Blueprints, ComfyUI)

| Паттерн | Описание | Применимость |
|---------|----------|-------------|
| **Порт ввода/вывода** | Нода имеет входы (слева) и выходы (справа) | Агент = нода: принимает задачи (in) → выдаёт результат (out) |
| **Цветовая кодировка портов** | Тип данных = цвет порта | Тип связи: command=cyan, sync=gray, twin=diamond |
| **Фиксированный направленный граф** | Сверху-вниз или слева-направо | Иерархия Стратегия→Исполнение идеально ложится в DAG |
| **Миникарта** | Обзор в углу для навигации | Уже есть, но нужно улучшить |
| **Zoom + Pan + Select** | Стандартная навигация | Есть, но нужно добавить drag нод |

### 2.2 Graph Visualization (D3, Cytoscape, Neo4j, vis.js)

| Паттерн | Описание | Применимость |
|---------|----------|-------------|
| **Force-directed layout** | Автоматическая укладка с отталкиванием | Ключевое: решает проблему наложений |
| **Hierarchical (Dagre/Sugiyama)** | Слойная укладка DAG | Лучший выбор для подчинения parentId |
| **Radial tree** | Дерево от центра | Подходит если 1 корневой агент |
| **Clique detection + grouping** | Авто-обнаружение кластеров | Группировка по roleGroup с сворачиванием |
| **Edge bundling** | Объединение параллельных связей | Упрощает визуальный шум от sync-связей |

### 2.3 Dashboard Design (Dribbble, Figma community)

| Паттерн | Описание |
|---------|----------|
| **Split panel** | Граф слева + детальная панель справа |
| **Toolbar top** | Фильтры, search, view mode в хедере |
| **KPI strip** | Статистика в верхней полосе (total/active/error) |
| **Breadcrumb navigation** | Путь по иерархии при drill-down |

## 3. Предлагаемая архитектура Hierarchy v2

### 3.1 Layout: Dagre (иерархический направленный граф)

Заменить radial на **Dagre/Sugiyama** — стандартную алгоритмическую укладку для DAG:
- Корневые агенты (Стратегия) наверху
- Исполнение внизу
- Связи parentId = вертикальные рёбра
- Sync/twin = горизонтальные/диагональные

### 3.2 React Flow как базовый фреймворк

React Flow (reactflow.dev) даёт из коробки:
- Drag-and-drop нод
- Zoom/Pan с momentum
- Minimap
- Custom node components
- Edge types с анимацией
- Layouting через dagre/elkjs

### 3.4 Ключевые изменения

| Было | Станет |
|------|--------|
| Custom SVG + Canvas | React Flow (reactflow) |
| Radial concentric rings | Dagre hierarchical layout |
| Статические позиции | Drag-and-drop + auto-layout |
